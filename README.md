# 🃏 Custom GUI UNO Game (Java Swing Framework)

This project extends an existing **console-based UNO game** by introducing a **custom graphical user interface (GUI) framework** built on top of **Java Swing**.  
The main goal is to encapsulate core GUI concepts—**components, events, and layout management**—into a **simplified, reusable framework** that abstracts away direct Swing usage.

---

## 🎯 Project Objective
The project aims to:
- Reinforce understanding of **GUI architecture** and **component-based design**.
- Build a **custom GUI layer** that acts as a framework on top of Swing.
- Re-implement the UNO game using **only custom components**, without directly interacting with Swing classes.
- Demonstrate principles of **abstraction**, **event-driven programming**, and **software modularity**.

---

## 🧩 Key Features
- ✅ Full UNO game logic (deck, rules, turns, actions)
- 🧱 Custom GUI framework (extends Swing)
- 🎨 Reusable components (buttons, panels, labels, etc.)
- ⚡ Event system for handling user interactions
- 🔁 Dynamic updates and real-time player feedback
- 🧩 Clean, object-oriented design promoting abstraction

---

## 🧱 Technologies
- **Language:** Java (JDK 8+)
- **Framework:** Custom GUI layer built over **Java Swing**
- **Paradigm:** Object-Oriented Programming (OOP)
- **Architecture:** MVC-style separation between game logic and GUI

---

## 🎮 Gameplay Overview

- Players take turns matching the color or symbol of the top card.
- Action cards (Skip, Reverse, Draw Two, Wild, Wild Draw Four) modify the flow.
- The GUI dynamically updates hands, deck, and game states.
- The first player to discard all their cards wins!
